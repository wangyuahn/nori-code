# Nori Work v1.0.0-pre.2 变更整理

> 状态：已实现并完成 Windows 安装包构建。版本为 `v1.0.0-pre.2`，以已发布 Git 标签 `v1.0.0-pre.1`（提交 `88ce86f`）为基线，记录截至 2026-07-17 的预发布改动。

## 修复的问题

### Plan / 执行模式切换延迟

模式按钮原本优先读取 `sessionStatus.plan_mode`。该状态来自会话状态轮询，空闲时约每 3 秒刷新一次，因此它会暂时覆盖 `useSessions` 已经写入的乐观更新，表现为点击后需要等待数秒才切换。

现在按钮和 `Tab` 快捷键会先更新本地显示，再异步保存会话配置；服务端状态确认后，本地临时状态自动清除。如果保存失败，界面会回到服务端状态。事实路径：

- `apps/nori-web/src/components/ChatView.tsx`
- `apps/nori-web/test/ChatView.test.ts`

### 智能体协作入口状态不一致

有 Agent 运行时，智能体协作入口此前存在三种不一致：离开协作页时显示黄色，位于协作页时变成绿色；浅色主题的高优先级活动样式可能覆盖黄色；如果页面错过 `subagent.started` WebSocket 事件，协作页仍能通过后台任务接口看到 Agent，但导航入口不会变黄。

现在所有活动 Agent 都使用同一黄色状态和数量徽标，不受当前页面或明暗主题影响。当前会话的后台 Agent 轮询结果也会参与全局计数，并与 Swarm 实时事件去重，避免同一个 Agent 被重复计算。事实路径：

- `apps/nori-web/src/App.tsx`
- `apps/nori-web/src/components/SwarmPanel.tsx`
- `apps/nori-web/src/styles/nori-theme.css`
- `apps/nori-web/test/PrimaryNavigation.test.ts`

### 自定义 Agent 在协作面板中的身份与输出验证

检查确认自定义 Profile 原本不会被 Swarm 状态投影过滤，流式 `assistant.delta`、最终输出和 token 也会进入任务卡片。本次在此基础上补充了可见 Profile 名称、后台状态恢复和服务端投影测试，防止后续把自定义 Agent 误当作内置 Agent 或丢失输出。

这部分属于显示增强和防回归加固，不把原本已经正常的消息链路误写成已修复缺陷。事实路径：

- `apps/nori-web/src/components/SwarmPanel.tsx`
- `apps/nori-web/test/SwarmPanel.test.ts`
- `packages/server/test/services.test.ts`

### 嵌套子 Agent 唤醒后仍显示等待中

子 Agent 调用下一层 Agent 后，父节点可能被投影为等待或暂停。嵌套任务完成时，运行时会通过新的 `turn.started` 轮次重新唤醒父 Agent，但协作状态投影此前只处理首次 `subagent.started`，没有把后续轮次启动映射回运行中。

现在任意已登记子 Agent 的新轮次启动都会立即把对应任务恢复为运行中，直到它再次等待或进入终态。事实路径：

- `packages/server/src/services/gateway/wsBroadcastService.ts`
- `packages/server/test/services.test.ts`

### 已有会话无法使用新建的自定义 Agent

全局配置保存后，已打开 Session 与 Agent 原本继续持有旧的 `customAgents` 配置对象，只有新建会话能看到新增角色。现在配置更新会原地同步到所有活动 Session，重建现有 Agent 的内置协作工具，并重新渲染 system prompt 中的 `<available_custom_agents>`，不需要重载会话或中断后台任务。事实路径：

- `packages/agent-core/src/rpc/core-impl.ts`
- `packages/agent-core/src/session/index.ts`
- `packages/agent-core/src/agent/index.ts`
- `packages/agent-core/test/rpc/config-rpc.test.ts`
- `packages/agent-core/test/session/init.test.ts`

### 手动终止 Agent 后协作入口仍显示黄色

前端此前只在 `subagent.completed`、`subagent.failed` 和 `subagent.suspended` 时清理实时 Agent ID，没有处理 `background.task.terminated`。同时，已停止 Swarm 的 Agent ID 离开活动 Swarm 集合后会被误算成普通后台 Agent。现在终止事件会立即移除对应 Agent，并使用当前 Session 的全部 Swarm 记录识别归属，只对仍运行或暂停的 Swarm 计数。事实路径：

- `apps/nori-web/src/hooks/useChatMessages.ts`
- `apps/nori-web/src/App.tsx`
- `apps/nori-web/test/useChatMessages.test.ts`
- `apps/nori-web/test/PrimaryNavigation.test.ts`

### 打包版 Pyright LSP 无法启动

原生 SEA 中未包含 Pyright 发布包，且打包后的 `nori.exe` 不能像普通 Node 一样执行语言服务器脚本，因此会报 `Cannot find module 'pyright/package.json'`。现在原生资源清单完整收集 Pyright 入口、运行时 chunk 和 typeshed fallback；Nori Work 启动后台服务时提供 Electron Node runtime，LSP 子进程使用 `ELECTRON_RUN_AS_NODE=1` 执行 Pyright。最终 `win-unpacked` 产物已实际完成 LSP `initialize` 握手并返回 capabilities。事实路径：

- `apps/nori-code/scripts/native/assets.mjs`
- `apps/nori-code/scripts/native/native-deps.mjs`
- `apps/nori-code/src/native/module-hook.ts`
- `apps/nori-desktop/src/main/ensure-server.ts`
- `packages/agent-core/src/services/lsp/lspService.ts`

### 流式 Markdown 文本选区扩大或消失

模型流式输出时，Markdown 每次更新都会替换整棵渲染 DOM，导致鼠标拖选中的文本节点失效，表现为选区突然扩大或在复制前消失。现在用户开始拖选时会冻结对应 Markdown DOM；松开鼠标后只要选区仍存在就继续冻结，取消选区后再一次追上最新输出。代码复制按钮也会等选区释放后挂载，避免在流结束瞬间扰动代码块选区。事实路径：

- `apps/nori-web/src/components/MarkdownView.tsx`
- `apps/nori-web/test/MarkdownView.test.ts`

## 新增功能

### 自定义 Agent 独立模型

每个自定义 Agent 现在可以指定模型别名，也可以保留为空并继承父 Agent 模型。智能体协作页面的编辑器会列出当前模型目录，并显示已配置模型的名称和 Provider；配置中已保存但当前不可用的模型仍会保留显示，不会静默丢失。

模型配置会写入 `custom_agents.<name>.model`，并出现在提供给主 Agent 的自定义 Agent 列表中。新建子 Agent 和恢复已有子 Agent 时都会重新应用该 Profile 的模型；未指定模型的自定义 Agent 及内置 Agent 继续继承父 Agent 当前模型。事实路径：

- `packages/agent-core/src/config/schema.ts`
- `packages/agent-core/src/profile/custom.ts`
- `packages/agent-core/src/profile/types.ts`
- `packages/agent-core/src/session/subagent-host.ts`
- `apps/nori-web/src/components/SwarmPanel.tsx`

### Markdown 数学公式渲染

共用 `MarkdownView` 已接入本地 KaTeX，支持行内 `$...$` 和块级 `$$...$$` 公式，也支持公式紧邻中英文文本。代码块内的美元符号不会被当作公式。

AI 回答、实时输出、Markdown 文件预览、记忆/知识库笔记、Agent 输出、规划审批和问题说明都复用同一个组件，因此公式解析、安全清理和代码复制逻辑没有重复实现。KaTeX 样式与字体会随 Web/桌面构建打包，不依赖 CDN。事实路径：

- `apps/nori-web/src/components/MarkdownView.tsx`
- `apps/nori-web/src/components/ChatView.tsx`
- `apps/nori-web/src/components/FilePreview.tsx`
- `apps/nori-web/src/components/VaultBrowser.tsx`
- `apps/nori-web/test/MarkdownView.test.ts`

### README 截图更新

`v1.0.0-pre.1` 之后的提交 `d577049` 更新了 `docs/images/nori-work.png`，README 继续引用同一路径，因此不需要修改文档中的图片链接。

## 配置与兼容性

- `custom_agents.<name>.model` 为可选字段，旧配置无需迁移。
- 未设置模型时保持原有的父 Agent 模型继承行为。
- 旧的 `nori-coder` Profile 兼容映射仍然保留。
- KaTeX 与 `marked-katex-extension` 是 Nori Web 的本地运行时依赖，锁文件已同步更新。
- Pyright 完整运行时作为 SEA 原生资源打包；桌面端通过 Electron Node runtime 启动该语言服务器。
- CLI 与桌面版本号已更新为 `1.0.0-pre.2`；该版本当前仅存在于本地工作树，尚未发布。
